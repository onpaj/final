# i18n Design — Czech Primary, English Fallback

**Date:** 2026-04-11
**Status:** Approved

## Goal

Localize all frontend UI strings to Czech. English strings remain as the fallback (if a key is missing in `cs/translation.json`, `i18next` falls back to `en/translation.json`). No language switcher — Czech is the fixed display language. Number and date formatting already uses `cs-CZ` locale throughout and is unchanged.

---

## Libraries

| Package | Version |
|---------|---------|
| `i18next` | latest |
| `react-i18next` | latest |
| `i18next-http-backend` | latest |

`i18next-http-backend` loads translation JSON files at runtime from `public/locales/` rather than bundling them, keeping the JS bundle lean.

---

## File Structure

```
frontend/
  public/
    locales/
      cs/
        translation.json    ← primary (Czech)
      en/
        translation.json    ← fallback (English)
  src/
    i18n.ts                 ← i18next init, imported once in main.tsx
```

---

## i18next Configuration (`src/i18n.ts`)

```ts
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import Backend from "i18next-http-backend";

i18n
  .use(Backend)
  .use(initReactI18next)
  .init({
    lng: "cs",
    fallbackLng: "en",
    backend: { loadPath: "/locales/{{lng}}/{{ns}}.json" },
    interpolation: { escapeValue: false },
  });

export default i18n;
```

Import in `main.tsx` before the React render call:

```ts
import "./i18n";
```

## Out of Scope

- No language toggle UI — Czech is fixed
- Category/group names come from the database and are not translated here
- Backend error messages are not translated (not shown directly in UI)
- Number and date formatting (`toLocaleString("cs-CZ")`) is already correct and unchanged
